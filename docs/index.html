Configure the project

  1. Ensure Cocoapods are installed.
  2. Run pod install in terminal.
  3. Open VISDK.xcworkspace (don’t use .xcodeproj)

Folder structure

  1. InternalClient - internal test app which use VISDK public api.
  2. SDK - VISDK sources
  3. Scripts ..* build.sh - builds VISDK scheme for device and simulator, then merges them into universal framework ..* submit - stolen from Fabric. Sends specified .ipa to the Fabric. Project structure

Pods are only used in unit tests. Try to avoid usage of side pods in VISDK source. Both ‘libVSDK’ and ‘VISDK’ targets are maintained. Currently ‘VISDK’ is used to build a framework. Unit test are assigned to ‘libVSDK’ historically.
